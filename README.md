# COMP3011 CW2 — Search Engine Tool

A production-quality search engine for [quotes.toscrape.com](https://quotes.toscrape.com/) built with Python, Requests, and BeautifulSoup.

## Features

| Feature | Description |
|---------|-------------|
| **Polite Crawler** | BFS crawl with 6-second delay, robots.txt awareness, exponential-backoff retries |
| **Inverted Index** | TF-IDF weighted, with word frequency, positions, and document metadata |
| **Compressed Storage** | gzip-compressed JSON with MD5 integrity checksums |
| **Ranked Search** | AND-logic multi-word search ranked by TF-IDF score |
| **Snippet Extraction** | Contextual excerpts with highlighted query terms |
| **Query Suggestions** | Levenshtein "did you mean?" for misspelled words |
| **Performance Benchmarking** | Search latency and index build-time tracking |
| **Professional Test Suite** | 50+ pytest tests with mocking, >85% coverage target |

## Usage

Run the interactive CLI from the project root directory:

```bash
python -m src.main
```

### Available Commands

| Command | Description |
|---------|-------------|
| `build` | Crawl quotes.toscrape.com, build inverted index, save to `data/index.json.gz` |
| `load` | Load a previously built index from disk |
| `print <word>` | Display the inverted-index entry for a word (frequency, TF, IDF, positions) |
| `find <query>` | Search for pages matching query terms (AND logic for multi-word) |
| `stats` | Show index statistics (terms, documents, build time) |
| `help` | Display available commands |
| `quit` / `exit` | Exit the program |

## Architecture & Design

### Crawler (`src/crawler.py`)
- **BFS traversal** starting from the base URL
- **URL normalisation** (strips fragments, query strings, trailing slashes)
- **Pattern-based link filtering** — only follows `/page/N/`, `/tag/xxx/`, `/author/xxx/`
- **robots.txt compliance** via `urllib.robotparser`
- **6-second politeness delay** enforced between requests
- **Exponential backoff** on transient errors (connection, timeout, 5xx)
- **No retry on 404** — immediately skips missing pages
- **Progress reporting** via Python logging

### Indexer (`src/indexer.py`)
- **Inverted index**: `word → {url → {frequency, positions, tf}}`
- **Document metadata**: URL, title, word count, raw text (for snippets)
- **Text normalisation**: lowercase, punctuation removal, contraction handling
- **Stop-word removal**: 150+ common English stop words excluded
- **TF-IDF scoring**: smoothed IDF formula `log((N+1)/(df+1)) + 1`
- **Compressed persistence**: gzip JSON with MD5 checksums for integrity

### Search Engine (`src/search.py`)
- **AND semantics** for multi-word queries
- **TF-IDF ranking** — sum of per-term scores per document
- **Snippet extraction** with contextual highlighting (`**term**`)
- **Query suggestions** via Levenshtein edit distance (O(V) scan)
- **Performance benchmarking** — latency recorded for each query

### Complexity Analysis

| Operation | Time Complexity | Space Complexity |
|-----------|----------------|-----------------|
| Crawl | O(N × D) | O(N × S) |
| Build index | O(N × W) | O(V × N) |
| Single-term search | O(D) | O(D) |
| Multi-term search | O(T × D) | O(D) |
| Save/Load | O(V × D) | O(V × D) |
| Query suggestion | O(V × L²) | O(L) |

Where: N = pages, D = politeness delay / matching docs, S = avg page size,
W = avg words/page, V = vocabulary size, T = query terms, L = word length.
